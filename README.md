## DDD勉強会の目的
* DDDを採用しているが、設計方針を立てることをコンサルタントに頼っている    
自分たちでAPアーキを組めるようになるため  

* 設計思想やオブジェクト思考設計本としてDDDは今の所優秀  
-> デベロッパーとしては知っておいてそんはない

---
DDDをやるべきは事業会社のエンジニア → もっとも優先すべき　
最新技術はコンサルタントに頼ってもいいかもしれないが
自分の担当するドメインのモデリングに関しては頼るべきではないと思った  

---
